# Runtime Behavior

## 概要

アプリケーションは、海洋ごみ回収ロボットと海洋生物の相互作用を 2D 連続座標空間でシミュレーションする。  
毎 tick ごとに観測、共有、移動、回収、帰還、再生成、スコア更新を行い、WebSocket で frontend に配信する。

## エージェント

### Scout

- ごみを検知する
- 検知結果を共有ターゲットへ登録する
- 初期実装では全体共有を使う
- エネルギーが減ると基地へ戻る

### Collector

- 共有ターゲット、または近傍ごみへ向かう
- ごみに接触すると回収状態になる
- 回収後は基地へ戻り、ごみ搬入でスコア加算する
- エネルギーが尽きても低速帰還モードで基地へ向かう

### Marine Life

- 近くのロボットから離れる
- ロボットに近づかれるほど stress が増える
- stress が閾値を超えると消滅する
- 消滅後は通常より遅い頻度で再生成される

### Trash

- 初期配置される
- 一定 tick ごとに追加生成される
- 緩やかに漂流する

## 基地

- 基地は単一で、画面下部の固定座標に置く
- 充電とごみ搬入は同じ基地で行う
- 低エネルギー時の帰還先でもある

## 1 tick の流れ

1. 必要なら新しいごみを生成する
2. Scout が周囲のごみを検知する
3. 検知済みごみ情報を共有ターゲットへ反映する
4. Collector が共有ターゲットや近傍ごみを追跡する
5. ごみ回収と基地搬入を処理する
6. ロボット同士の衝突を集計する
7. Marine Life の stress と再生成を更新する
8. スコアと統計を更新する
9. Snapshot を配信する

## スコア

- `trash_delivered` が増えるほど加点
- `collisions` が増えるほど減点
- `marine_life_stress` が高いほど減点
- `energy_remaining` が高いほど加点

## 衝突

- 衝突は半径ベースで判定する
- 初期実装では robot 同士の衝突のみをスコア対象とする

## フェーズ

- `idle`: 未開始
- `running`: 実行中
- `stopped`: 手動停止
- `completed`: step 上限到達
