# Runtime Behavior

## 概要

アプリケーションは、海洋ごみ回収ロボットと海洋生物の相互作用を 2D 連続座標空間でシミュレーションする。  
毎 tick ごとに観測、共有、移動、回収、帰還、再生成、スコア更新を行い、WebSocket で frontend に配信する。

## エージェント

### Scout

- ごみを検知する
- 検知結果を共有ターゲットへ登録する
- 初期実装では全体共有を使う
- エネルギーが減ると基地へ戻る

### Collector

- 共有ターゲット、または近傍ごみへ向かう
- ごみに接触すると回収状態になる
- 回収後は基地へ戻り、ごみ搬入でスコア加算する
- エネルギーが尽きても低速帰還モードで基地へ向かう
- シミュレーション時間が半分（`steps/2`）経過すると、自動・手動問わずすべての Collector の移動速度が 3 倍になる
- 基地で`enrgey`<`max_energy`の間は完全停止して充電する
- 自動 Collector は、他のロボットと衝突しても停止ペナルティは受けないが、全体スコアの `collisions` カウントは増加する
- `enable_manual_robot` が有効な場合、手動 Collector が1体追加される
- 手動 Collector は WebSocket の `manual_move` action で WASD / 矢印キー入力に追従する
- 手動 Collector は魚避けとロボット避けを行わず、衝突時は一定時間（`manual_penalty_ticks`）完全に停止するペナルティを受ける。ただし、ごみを回収して基地へ運搬中（満載状態）の場合はペナルティを免除される

### Marine Life

- Couzin 3-zone ルール(ZOR/ZOO/ZOA)で群れる
- `marine_life_avoid_radius` 内のロボットから離れる
- ロボットが回避半径へ入った瞬間をエッジ検知して `robot_fish_contacts` を加算する
- `fish_eat_radius` 内のごみを偶発的に食べ、ごみを消して `fish_ate_trash` を加算する
- 個体は消滅せず、`marine_life_count` を常時維持する

### Trash

- 初期配置される
- 一定 tick ごとに追加生成される
- 緩やかに漂流する

## 基地

- 基地は単一で、画面下部の固定座標に置く
- 充電とごみ搬入は同じ基地で行う
- 低エネルギー時の帰還先でもある

## 1 tick の流れ

1. 必要なら新しいごみを生成する
2. Trash が漂流する
3. Scout が周囲のごみを検知し、検知済みごみ情報を共有ターゲットへ反映する
4. Collector が共有ターゲットや近傍ごみを追跡する。手動 Collector は最新の入力方向へ移動する
5. Predator が Marine Life を追跡または巡回する
6. Marine Life の panic 判定、ロボット接触カウント、近傍ごみの誤飲処理を行う
7. ロボット同士の衝突を集計する
8. 基地での充電とごみ搬入を処理する
9. 無効になった共有ターゲットを削除する
10. スコアと統計を更新する
11. Snapshot を配信する

衝突集計は基地搬入より前に行う。  
これにより、手動 Collector が満載状態で基地へ戻った tick では、既存の「運搬中は衝突停止ペナルティを受けない」ルールが適用される。基地搬入で `carrying_trash_id` が消えた直後だけ停止ペナルティを受ける状態遷移は作らない。

## スコア

- `trash_delivered` が増えるほど加点
- `collisions` が増えるほど減点
- `energy_remaining` が高いほど加点

## 衝突

- 衝突は半径ベースで判定する
- 初期実装では robot 同士の衝突のみをスコア対象とする

## フェーズ

- `idle`: 未開始
- `running`: 実行中
- `stopped`: 手動停止
- `completed`: step 上限到達
