システム設計方針

１．このプロジェクトは、ogawa branchを作業している人と、このMainブランチを作った二名がプロマネ的な動きを行い、そのシステム全体を設計、修正するものである。
２．ほかに三名のプロジェクトメンバーがいて、基本要件定義および設計、木戸コードの構築が終わった後に、各々がrobot_logic.xmlの設計になるように微調整を加える（デザインの追加、モーションの追加、挙動およびパラメータの調整、各種コンポーネント的機能の追加）予定になっている。
３．２の要件を満たすため、DBやシステム内のデータ連携APIは各ロボットごとに統一し、オブジェクトやアクタのI/Oは共通であるか、それ専用の設計を施す必要がある。また、各ロボットの機能面についても、拡張性が高い設計にし、コンポーネント的に機能と追加、廃止、寮生できるものであり、各ロボットの独立性は適度に担保されている必要がある。

以上2点の実装方針を踏まえて、master_coduments.mdの基本構築を行う。
